가상 환경¶

virtualenv 에 대해서 들어본 적 있나요? 만약 당신이 python 을 시작하는 사람이라면 당신은 들어본 적이 없을 겁니다. 그러나, python 중급이상 개발자라면 virtualenv 는 여러분의 toolset 의 vital 부분입니다. 자 그럼 virtualenv 는 정말 무엇일까요? virtualenv 는 독립된 python 환경을 만들어주는 도구입니다. 여러분의 Application에서 LibraryBar 라는 python2 기반의 lib 가 필요하다고 상상해보세요. 그런데 또다른 Application 에서는 version3 기반의 lib를 필요로 하는 상황입니다. 당신은 이러한 application들을 어떻게 사용하고 개발하나요?

만약 /usr/lib/python2.7/site-packages (또는 여러분 플랫폼의 표준 위치가 어디이든지간에) 의 위치에 모든 것을 다 install 한다면, 업그레이드 하지 않아야 하는 package를 의도치 않게 업그레이드 하는 상황에 쉽게 처할 수 있습니다. 여러분은 완전히 개발된 application을 가지고 있다고 상상해보세요. 그리고 여러분은 application이 사용한 lib의 어떤 변경도 원치 않는 상황입니다. 하지만 동시에 여러분은 다른 Application을 개발하게 되었는데, 해당 어플리케이션은 이 lib 의 업데이트를 필요로 합니다. 당신은 어떻게 하시겠습니까? 이 때가 바로 virtualenv 가 필요한 때 입니다. 이는 여러분의 python 어플리케이션을 위한 독립된 개발환경을 제공해주고, 여러분이 global영역이 아닌 독립된 환경에 python lib 를 설치하도록 해줍니다.

virtualenv 의 설치를 위해 아래 명령어를 shell 에 입력해보세요.

$ pip install virtualenv

자 이제, 관련 명령어 몇개를 나열해 볼건데요. 가장 중요한 것은 아래 명령어들입니다.

  • $ virtualenv myproject
  • $ source bin/activate

첫번째 명령어는 myproject 폴더 내에서 독립된 virtualenv 개발환경을 만들어 줍니다. 그리고 두번째 명령어는 독립된 개발환경을 활성화해줍니다. 첫번째 명령어가 실행되는 동안, 당신은 결정해야 합니다.

virtualenv 가 system 영역의 site-packages 에서 패키지를 사용하기를 원하는지 virtualenv의 site-packages 에 모든 것을 설치하고 사용하길 원하는지 결정해야 합니다. 기본적으로는 virtualenv 는 global 영역의 site-packages 에 접근할 수 없습니다. 만약 여러분의 virtualenv 가 system 의 site-packages 의 접근을 원한다면 여러분의 virtualenv 를 생성할때 --system-site-packages 를 사용하여 교체하면 됩니다. 아래는 명령어 입니다.

$ virtualenv --system-site-packages mycoolproject

여러분은 env 가 동작하지 않도록 할 수 있습니다.

$ deactivate

Bonus

여러분은 bash 와 zsh 에서 사용가능한 lib인 smartcd 을 사용할 수 있습니다. 그리고 이는 여러분의 bash(또는 zsh) 환경에서 사용하던 cd를 대신합니다. 이것은 여러분이 directories 를 변경할 때, virtualenv 를 활성화 그리고 비활성화 하는데 정말 유용합니다. 나는 이를 꽤 사용해봤고 굉장히 좋아하는 기능입니다. 다음의 링크에서 좀 더 자세한 내용을 확인할 수 있습니다. GitHub
